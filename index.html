Guilherme Pontes

Quem sou eu?

Sou um profissional de TI, que adora TI, principalmente quando trata-se de minúcias relacionadas às áreas de infraestrutura e desenvolvimento.

Trabalho com Linux há muitos anos (não me lembro exatamente desde quando, mas é da época dos modems 56k e de quando precisávamos compilar o kernel para habilitar a escrita nas partições windows). Já tive oportunidade de usar as distribuições Conectiva, Redhat, Fedora, Mandriva, Debian, Ubuntu (esta é a que estou utilizando agora), Slack, Suse, Kurumin, FreeSCO, Coyote, entre outros.

Também tenho muito conhecimento em desenvolvimento de aplicações (talvez essa seja minha área predileta, aliás). Meus primeiros programas foram desenvolvidos na famosa, mas moribunda, linguagem Action Script do Flash (isso por volta de 1999 ou 2000).

Só quando entrei na faculdade que amadureci (ou não) meu conhecimento. Desde lá, venho desenvolvendo sistemas como profissional autônomo ou nas empresas/instituições em que trabalhei. Sei programar em Java, Python (Django ou Flask), PHP, C# (.Net - não tanto quanto Java), Delphi, C++, Matlab, JavaScript (considerando suas vertentes mais legais como JQuery, React, Angular e NodeJS), CSS e HTML.

Também tenho experiência com bancos Oracle, Firebird (antigo Interbase), MySQL, PostgreSQL e MongoDB. Tenho fluência em PL/SQL, Triggers, stored procedures e todo aparato técnico necessário para trabalhar com banco de dados.

Acho que não vem ao caso, mas já trabalhei com redes, montagem e manutenção (hardware), um pouco de design (Corel, CSS, Flash, etc), e desenvolvimento de jogos para celular e PC.

Igreja Presbiteriana de Jardim Primavera

CompDB

Sistema desenvolvido para realizar comparação (registros) entre os bancos DMS2 (banco hierárquico de Plataforma Alta da UNISYS) e Oracle. Esse sistema, complexamente falando, é bem interessante. Ele possui o módulo em Java com 5 threads simultâneas (sendo 4 para processamento e 1 para a interface), que é responsável pelo controle geral do aplicativo, pela conexão com o banco Oracle, e pela comparação dos registros das tabelas. Existe também o módulo em Visual Basic que conecta ao DMS2 via OleDB. Esses dois programas funcionam juntos, trocando informações entre si via rede localhost. Como ele foi desenvolvido para rodar em processadores Quad Core, as 4 threads que realizam a comparação dos registros consomem, cada uma, um núcleo específico.
